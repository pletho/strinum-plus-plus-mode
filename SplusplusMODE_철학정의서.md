# Strinum 생태계 – 공통 철학 선언
버전: 1.0
상태: 공개 선언

## 1. 근본 원칙

Strinum 생태계는 다음의 믿음 위에 존재한다:

> 의미가 구조보다 먼저 존재한다.  
> 구조가 실행보다 먼저 존재한다.  
> 실행은 반드시 조직되어야 한다.

이 생태계는 단순한 프레임워크가 아니다.  
AI 시대의 실행 철학 체계이다.

---

## 2. 공통 사상

### 2.1 도메인 우선주의

모든 시스템은 코드가 아니라 도메인 의미에서 출발해야 한다.

### 2.2 구조 중심주의

명확한 구조 없이 실행은 존재할 수 없다.

### 2.3 프레임 내부 실행 원칙

모든 실행은 정의된 프레임 안에서만 발생한다.

### 2.4 조직화된 지능

AI는 대화형 도구가 아니라 계약 기반 실행 조직으로 동작해야 한다.

---

## 3. 공개 정책

철학과 구조 사상은 공개한다.  
결정적 실행 엔진은 보호될 수 있다.

---

## 4. 생태계 핵심 문장

SUL은 의미를 정의한다.  
Strinum은 구조를 정의한다.  
FrameSet은 존재 방식을 정의한다.  
S++MODE는 조직 방식을 정의한다.

# Strinum++MODE (S++MODE)
버전: 1.0
상태: 조직 모델 공개

## 1. 정의

S++MODE는 AI 기반 실행 조직 모델이다.

전체 생태계를 계약 기반 세션으로 통제한다.

---

## 2. 목적

- AI 오케스트레이션
- 병렬 세션 관리
- 결정적 병합 제어
- 계약 기반 검증

---

## 3. 철학

AI는 단순 대화 도구가 아니다.  
구조화된 팀으로 동작해야 한다.

---

## 4. 생태계 위치

S++MODE는

SUL → Strinum → FrameSet → Runtime

전 계층을 통제하는 메타 구조이다.
